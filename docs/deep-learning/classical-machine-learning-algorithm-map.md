# 经典机器学习算法地图：从概率、距离、边界到集成学习

*Yohan Lab · 机器学习算法板块（二）*

> 第一阶段 D 级冲刺笔记 · 2026.05.23

> **本文定位**
>
> 这篇笔记整理的是经典机器学习后半部分的核心算法。目标不是背诵名词，而是用第一性原理理解每个模型为什么存在、解决什么问题、如何工作、适合什么场景，以及它和其他模型的边界在哪里。

## 一、总览：经典机器学习的几种基本思维

本阶段学习可以压缩为五种建模思维：概率、距离、边界、树结构、集成与降维。每一类算法都不是孤立名词，而是在回答不同类型的问题。

| 建模思维 | 代表算法 | 第一性原理 |
| --- | --- | --- |
| 概率思维 | Naive Bayes | 在不确定信息中，根据已有证据计算类别概率。 |
| 距离思维 | KNN / K-means | 相似样本靠得更近；分类靠邻居，聚类靠中心。 |
| 边界思维 | SVM | 寻找能最大化安全间隔的分类边界。 |
| 规则思维 | Decision Tree | 把复杂判断拆成一系列 if-else 规则。 |
| 集成思维 | Random Forest / Boosting / XGBoost / LightGBM / CatBoost | 让多个弱模型合作，降低偏差或方差。 |
| 压缩思维 | PCA | 在高维数据中找到最能保留信息的方向。 |

## 二、核心算法模块

### 01. Naive Bayes：用简单假设换取高效概率分类

| 第一性原理 | 在看到一组特征后，判断样本最可能属于哪个类别。 |
| --- | --- |
| 核心机制 | 使用贝叶斯公式计算后验概率，并假设在给定类别的前提下，各特征条件独立。这个假设并不代表世界真的独立，而是主动简化复杂关系。 |
| 输入 / 输出 | 输入是特征集合，输出是类别概率或最终类别。典型任务包括垃圾邮件分类、文本分类、情感分类。 |
| 优势 | 训练快、推理快、小数据也能工作，适合作为文本分类和概率分类的快速基线。 |
| 局限 | 条件独立假设过强；当特征之间存在强相关时，概率估计可能失真。 |
| 面试级一句话 | 朴素贝叶斯不是相信世界简单，而是用一个简单假设换取高效、可解释的分类。 |

### 02. KNN：有标签的邻居投票

| 第一性原理 | 相似样本在特征空间中应该彼此接近，因此新样本可以由附近样本共同判断。 |
| --- | --- |
| 核心机制 | 计算新样本与训练样本的距离，选取最近的 K 个邻居，再根据邻居标签投票或加权投票。 |
| 输入 / 输出 | 输入是带标签训练样本和新样本特征，输出是新样本类别或数值预测。 |
| 优势 | 几乎不需要训练，直观、易实现，适合小规模、低维、距离度量有意义的数据。 |
| 局限 | 预测阶段慢；K 值敏感；受噪声、异常点和特征尺度影响；高维空间中距离会失去区分力，这就是维度诅咒。 |
| 面试级一句话 | KNN 是近朱者赤的算法：K 太小容易偏激，K 太大容易被多数类稀释。 |

> **关键校正：维度诅咒**
>
> 高维数据中，样本之间的距离会变得不稳定，最近和最远的差异可能不再明显。KNN 依赖距离判断相似性，因此在高维空间中特别容易失效。

### 03. SVM：寻找最大安全间隔的分类边界

| 第一性原理 | 分类边界不仅要能分开样本，还要尽量离两类样本都远，从而提升泛化能力。 |
| --- | --- |
| 核心机制 | SVM 寻找最大间隔超平面；真正决定边界的是离边界最近的关键样本，即支持向量。通过核函数可以在不显式展开高维特征的情况下处理非线性问题。 |
| 输入 / 输出 | 输入是特征向量与标签，输出是分类边界和样本类别。 |
| 优势 | 适合中小规模、高维、边界相对清晰的分类任务；支持向量使模型关注关键样本。 |
| 局限 | 大规模数据训练成本高；核函数和参数选择影响大；结果解释性有限。 |
| 面试级一句话 | SVM 的核心不是找到一条线，而是找到最稳、最有安全距离的边界。 |

### 04. Decision Tree：把判断拆成规则路径

| 第一性原理 | 复杂决策可以拆成一系列局部判断，每个节点只回答一个问题。 |
| --- | --- |
| 核心机制 | 通过特征分裂不断划分数据，使每个叶子节点中的样本尽可能纯。整个模型等价于一系列 if-else 规则。 |
| 输入 / 输出 | 输入是样本特征，输出是类别或回归值。 |
| 优势 | 可解释性强，容易可视化，适合表格数据，也能处理非线性关系。 |
| 局限 | 树太深会记住训练集噪声，导致过拟合。因此需要限制最大深度、最小样本数或进行剪枝。 |
| 面试级一句话 | 决策树的强项是清晰，弱点也是清晰：它容易把偶然噪声也切成规则。 |

### 05. Random Forest：Bagging 与集体智慧

| 第一性原理 | 单棵决策树容易偏激，多棵不同视角的树共同判断可以降低不稳定性。 |
| --- | --- |
| 核心机制 | 通过随机抽样数据和随机选择特征，训练多棵决策树，再用投票或平均得到最终结果。这属于 Bagging 思想。 |
| 输入 / 输出 | 输入是结构化特征与标签，输出是集成后的分类或回归结果。 |
| 优势 | 稳定性强，抗过拟合能力优于单棵树，适合表格数据，并可输出特征重要性。 |
| 局限 | 解释性不如单棵树；模型更大；在高维稀疏文本任务中未必是最佳选择。 |
| 面试级一句话 | Random Forest 是并行独立训练的集体投票，用多样性降低单棵树的偏差。 |

### 06. Boosting / XGBoost：串行纠错与残差逼近

| 第一性原理 | 一个弱模型不足以解决复杂问题，但多个弱模型可以按顺序纠错，逐步逼近正确答案。 |
| --- | --- |
| 核心机制 | Boosting 串行训练模型，每一轮重点修正前一轮的错误。GBDT 拟合残差；XGBoost 在优化中使用一阶和二阶梯度信息，使分裂和参数更新更精细。 |
| 输入 / 输出 | 输入是结构化数据与标签，输出是多个弱模型叠加后的强预测器。 |
| 优势 | 在表格数据、比赛和工业建模中表现强；能处理复杂非线性关系；预测能力强。 |
| 局限 | 参数较多；训练更复杂；若控制不好仍可能过拟合。 |
| 面试级一句话 | 如果 Bagging 是并行投票，Boosting 就是串行复盘错题。 |

### 07. LightGBM：用工程优化加速 GBDT

| 第一性原理 | 在大规模数据上，模型不仅要准，还要能高效训练。 |
| --- | --- |
| 核心机制 | LightGBM 通过 histogram binning 将连续特征压缩成有限个区间，在桶级别寻找分裂点，牺牲少量精度换取速度和内存效率；同时引入 GOSS 和 EFB 等优化。 |
| 输入 / 输出 | 输入是大规模结构化数据，输出是梯度提升树模型。 |
| 优势 | 训练速度快，内存占用低，适合大规模表格数据。 |
| 局限 | 直方图离散化可能损失少量精度；参数设置不当仍可能过拟合。 |
| 面试级一句话 | LightGBM 的核心是以极小精度代价换取大规模训练效率。 |

> **补充：bin、EFB 与 GOSS**
>
> Histogram binning 是把连续数值压缩成若干区间。EFB 将互斥的稀疏特征打包以降低维度。GOSS 保留大梯度样本并采样小梯度样本，从而提升训练效率。三者都是 LightGBM 的工程优化，但作用层面不同。

### 08. CatBoost：类别特征与 Ordered Boosting

| 第一性原理 | 现实表格数据中存在大量类别特征，如何稳定处理这些特征并避免数据泄露，是提升泛化能力的关键。 |
| --- | --- |
| 核心机制 | CatBoost 是 GBDT 家族中的强模型，擅长 categorical features，并通过 Ordered Boosting 等机制减少 target leakage 和 prediction shift。 |
| 输入 / 输出 | 输入是结构化表格数据，尤其适合含类别特征的数据，输出是分类或回归结果。 |
| 优势 | 类别特征处理强，默认效果稳定，调参压力较低，泛化性能好。 |
| 局限 | 训练机制较复杂；在某些极大规模任务中需要权衡训练速度和资源。 |
| 面试级一句话 | CatBoost 不只是为了解决过拟合，而是为类别特征和训练偏差问题提供更稳的 GBDT 解法。 |

### 09. K-means：无标签的中心聚类

| 第一性原理 | 当没有标签时，仍然可以根据样本之间的相似性，把数据自动组织成若干组。 |
| --- | --- |
| 核心机制 | 预设 K 个簇中心，将每个样本分配到最近中心，再反复更新中心位置直到稳定。 |
| 输入 / 输出 | 输入是无标签样本特征，输出是每个样本的簇编号和簇中心。 |
| 优势 | 简单高效，适合用户分群、数据探索、图像压缩和初步聚类。 |
| 局限 | 必须提前指定 K；对初始中心和异常值敏感；对月牙形、非球状或密度差异大的数据效果较差。 |
| 面试级一句话 | K-means 是让无标签数据围绕 K 个中心自动站队。 |

### 10. PCA：寻找保留最多信息的视角

| 第一性原理 | 高维数据中并不是每个方向都同样重要，信息最多的方向通常表现为方差最大。 |
| --- | --- |
| 核心机制 | PCA 寻找数据方差最大的方向作为主成分，并用少数主成分替代原始高维特征。 |
| 输入 / 输出 | 输入是高维特征矩阵，输出是降维后的特征表示。 |
| 优势 | 适合降维、可视化、去噪和压缩特征，也能缓解高维问题。 |
| 局限 | 主成分不一定容易解释；只捕捉线性结构；降维会丢失部分信息。 |
| 面试级一句话 | PCA 像选择一个最好的拍照角度，用最少维度保留最多变化。 |

## 三、关键辨析：KNN 与 K-means

这两个算法名字相似，但任务完全不同。KNN 是监督学习中的分类/回归方法，K-means 是无监督学习中的聚类方法。

| 维度 | KNN | K-means | 一句话区分 |
| --- | --- | --- | --- |
| 学习类型 | 监督学习 | 无监督学习 | KNN 有标签，K-means 无标签 |
| K 的含义 | 邻居数量 | 簇的数量 | 一个看邻居，一个定组织数 |
| 核心任务 | 判断新样本类别 | 把数据自动分组 | 一个做预测，一个做结构发现 |
| 核心机制 | 邻居投票 | 中心迭代 | 一个靠附近样本，一个靠中心点 |

## 四、模型评估与泛化意识

模型不能只看“效果好不好”，必须说明好在哪里、坏在哪里、是否能泛化到新数据。评估指标是把直觉变成可比较证据的工具。

| 概念 | 第一性原理 |
| --- | --- |
| Accuracy | 整体预测正确率，适合类别较均衡的任务。 |
| Precision | 预测为正的样本中有多少是真的，适合误报成本高的场景。 |
| Recall | 真实正类中有多少被找回，适合漏报成本高的场景。 |
| F1-score | Precision 与 Recall 的折中。 |
| ROC-AUC | 衡量模型区分正负样本的排序能力。 |
| MAE / RMSE | 回归任务误差指标，RMSE 对大误差更敏感。 |
| Overfitting | 训练集表现好、新数据表现差，通常是方差过高。 |
| Underfitting | 训练集和测试集都表现差，通常是偏差过高。 |
| Bias-Variance Tradeoff | 模型太简单会偏差高，太复杂会方差高，训练目标是在二者之间寻找平衡。 |

## 五、本阶段压缩总结

本阶段学习的核心收获可以压缩为以下几句话：

- Naive Bayes 用条件独立假设把复杂证据拆成概率贡献。

- KNN 是有标签的邻居投票，适合小规模、低维、距离有意义的数据。

- SVM 寻找最大间隔超平面，支持向量决定分类边界。

- Decision Tree 通过规则路径解释决策，但树太深容易过拟合。

- Random Forest 是 Bagging：并行训练多棵树，用集体智慧降低不稳定性。

- Boosting 是串行纠错：后续模型不断修正前面模型的错误。

- XGBoost 强在梯度提升的工程优化与二阶信息利用。

- LightGBM 用 histogram binning、GOSS、EFB 等手段加速大规模训练。

- CatBoost 强在类别特征处理与 Ordered Boosting，能减少泄露和训练偏差。

- K-means 是无标签中心聚类，PCA 是保留最大方差方向的降维方法。

- 模型评估的核心不是报一个分数，而是解释错误类型、泛化能力和任务代价。

D 级的任务是建立机器学习基础地图。下一步进入 C 级：AI for Science 岗位知识地图。重点不再是泛学算法，而是将模型能力迁移到科学问题中。

- AI for Science：AI 如何进入科学发现流程。

- World Model：如何学习系统状态与演化规律。

- Scientific Agent：如何把文献、代码、实验和报告串成自动化科研工作流。

- AlphaFold / RoseTTAFold：蛋白质结构预测的代表工作。

- DiffDock：分子对接与药物发现的代表实操项目。

- GNN for Molecules：分子图、原子节点、化学键边与 message passing。

- GNoME：图网络与材料发现。
